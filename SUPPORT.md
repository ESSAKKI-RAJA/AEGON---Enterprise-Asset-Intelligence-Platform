# Support

This document explains how to get help with the AEGON platform, report issues, and request new features.

## Bug Reports and Feature Requests

The primary channel for bugs and feature proposals is the GitHub Issue Tracker.

- **Bug Report**: Use the [Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md) to describe the issue, reproduction steps, and your environment.
- **Feature Request**: Use the [Feature Request template](.github/ISSUE_TEMPLATE/feature_request.md) to describe the problem you are solving and your proposed solution.
- **General Question**: Use the [Question template](.github/ISSUE_TEMPLATE/question.md) for architecture, development, or configuration questions.

Please search for existing issues before opening a new one.

## GitHub Discussions

For broader topics — design decisions, architectural proposals, community feedback, or general announcements — use [GitHub Discussions](../../discussions).

Discussions are the preferred space for exploratory conversations that are not yet ready to be concrete issues.

## Security Vulnerabilities

**Do not report security vulnerabilities through public GitHub issues.**

Please review the [Security Policy](SECURITY.md) for responsible disclosure procedures and contact information.

## Contributing

If you would like to contribute code, documentation, or test coverage, please review [CONTRIBUTING.md](CONTRIBUTING.md) for the development workflow, architectural rules, and Definition of Done.

## Enterprise and Commercial Support

For enterprise deployment support, custom integrations, or professional services, contact the core team at `enterprise@aegon-enterprise.example.com`.
